## 2. Ajustar `Get-FilenameFromMetadata` para usar comentários “comuns”

Agora vamos estender `Get-FilenameFromMetadata` para, depois de tentar JSON e `@FILE:`, tentar também qualquer comentário que contenha `nome.ext`.

Substitua a função inteira por esta:

```powershell
function Get-FilenameFromMetadata {
    param(
        [string]$Content,
        [switch]$Verbose
    )

    if ($Verbose) { Write-Host "Buscando metadata JSON (\"filename\")..." -ForegroundColor Yellow }
    $fn = Extract-JsonMetadata $Content
    if ($fn) {
        if ($Verbose) { Write-Host "✓ JSON: $fn" -ForegroundColor Green }
        Write-Log "Nome detectado via JSON: $fn"
        return $fn
    }

    if ($Verbose) { Write-Host "Buscando @FILE nos comentários..." -ForegroundColor Yellow }
    $allFromComments = Extract-CommentMetadata $Content
    if ($allFromComments -and $allFromComments.Count -gt 0) {
        $chosen = Select-FilenameFromList -Names $allFromComments
        if ($chosen) {
            if ($Verbose) { Write-Host "✓ Comentário (@FILE): $chosen" -ForegroundColor Green }
            Write-Log "Nome selecionado via @FILE: $chosen"
            return $chosen
        } else {
            if ($Verbose) { Write-Host "Nenhum nome selecionado a partir de @FILE." -ForegroundColor Yellow }
        }
    }

    # NOVO: tentar qualquer comentário que contenha nome.ext
    if ($Verbose) { Write-Host "Buscando nome em comentários gerais (#, REM, ::, //, ;)..." -ForegroundColor Yellow }
    $allFromAnyComments = Extract-AnyCommentFilename $Content
    if ($allFromAnyComments -and $allFromAnyComments.Count -gt 0) {
        $chosen2 = Select-FilenameFromList -Names $allFromAnyComments
        if ($chosen2) {
            if ($Verbose) { Write-Host "✓ Comentário (genérico): $chosen2" -ForegroundColor Green }
            Write-Log "Nome selecionado via comentário genérico: $chosen2"
            return $chosen2
        } else {
            if ($Verbose) { Write-Host "Nenhum nome selecionado a partir de comentários genéricos." -ForegroundColor Yellow }
        }
    }

    return $null
}
```

Repare na ordem:

1. `"filename"` em JSON.
2. `@FILE:` (metadado explícito).
3. Qualquer comentário comum com `nome.ext` (inclui `#`, `REM`, `::`, etc.). [reddit](https://www.reddit.com/r/PowerShell/comments/1e75ybw/how_to_search_one_txt_for_filenames_a_second_for/)

## 3. Fluxo principal continua igual

No fluxo onde você chamou:

```powershell
$filename = Get-FilenameFallbackWithTimestamp -Content $content -Verbose
```

não precisa mudar: `Get-FilenameFallbackWithTimestamp` já faz:

- Chama `Get-FilenameFromMetadata` (que agora inclui JSON, `@FILE:` e comentários comuns).
- Se nada for encontrado, gera `script_YYYYMMDD_HHMMSS.ext`.

Com esse ajuste, os casos passam a funcionar assim:

### Python com shebang

```python
#!/usr/bin/python3
# meu_script.py
print("Hello")
```

- Não tem header na primeira linha.
- Não tem `"filename"` nem `@FILE:`.
- Mas tem um comentário `# meu_script.py`.  
  → `Extract-AnyCommentFilename` acha `meu_script.py`, mostra menu se tiver mais de um e usa esse nome.

### Batch com `REM` ou `::`

```bat
REM meu_script.bat
@echo off
echo Hello
```

ou

```bat
:: meu_script.bat
@echo off
echo Hello
```

- Pode ou não ser header (dependendo da função de header).
- Mesmo que não trate como header, `Extract-AnyCommentFilename` acha `meu_script.bat` em comentários e usa o nome.

Assim, você mantém:

- Primeira linha como **header especial** quando quiser controlar remoção/inserção de linha.
- Comentários em qualquer lugar como **fonte de nome** “comum”, sem a exigência de `@FILE:`.
